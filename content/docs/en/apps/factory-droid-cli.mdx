    Droid CLI is a command-line tool developed by Factory AI, designed to run as an AI software engineering agent. It allows users to interact with various large language models through the terminal, build, debug, and refactor code, and even create complete applications.

## AI Model Configuration Method

### Windows Graphical Guide

#### 1. Open Terminal

![windows_open_terminal](/assets/factory_droid_cli/windows_open_terminal.png)

#### 2. Install Factory Droid CLI

Official one-click installation command:

<Callout type="info" title="One-click Installation Command">
  ` irm https://app.factory.ai/cli/windows | iex `
</Callout>

![windows-install](/assets/factory_droid_cli/windows_install_droid.webp)

#### 3. Modify Configuration File

Droid CLI requires modifying the configuration file to use third-party APIs.

![windows-configure.webp](/assets/factory_droid_cli/factory_cli_setup_windows.png)

<Callout type="info" title="Modify Environment Variables">
  ` iex (irm
  'https://raw.githubusercontent.com/QuantumNous/new-api-docs/refs/heads/main/helper/factory-cli-setup.ps1')
  `
</Callout>

#### 4. Start Using Droid CLI

Now you can start using Droid CLI!

<Callout type="info" title="Launch Droid CLI">
  Launch Droid CLI directly: ` # Launch Droid CLI droid ` Use in a specific project: ` #
  Navigate to your project directory cd C:\path\to\your\project ` ` # Launch Droid CLI droid ` Press Enter to launch Droid CLI
</Callout>

Droid CLI requires users to log in to an official account (free) before use.

#### 5. Windows Common Issues Solution

<Callout type="info" title='Installation prompts "permission denied" error' collapsible>
  This is usually a permission issue. Try the following solutions:
</Callout>

    - Run `PowerShell` as an administrator

    - Or configure `npm` to use the user directory: `npm config set prefix %APPDATA%\npm`

<Callout type="info" title="PowerShell Execution Policy Error" collapsible>
  If you encounter an execution policy restriction, run: ` Set-ExecutionPolicy -ExecutionPolicy
  RemoteSigned -Scope CurrentUser `
</Callout>

### macOS/Linux Graphical Guide

#### 1. Install Droid CLI

<Callout type="info" title="Install Droid CLI">
  Open your terminal and run the following command: ` curl -fsSL https://app.factory.ai/cli | sh `
</Callout>

![macos-open-terminal](/assets/factory_droid_cli/macos_install_droid.webp)

![macos-img-02.webp](/assets/factory_droid_cli/macos_install_droid_2.webp)

Follow the installation prompts to modify environment variables (copy the installation prompt code directly):

For Linux, choose `~/.bashrc` or `~/.zshrc` as appropriate.

<Callout type="info" title="Droid CLI Environment Variables (Example Only)">
  ` echo 'export PATH=/Users/modify_here/.local/bin:$PATH' >> ~/.zshrc ` ` source
  ~/.zshrc `
</Callout>

#### 2. Modify Configuration File

Droid CLI requires modifying the configuration file to use third-party APIs.

<Callout type="info" title="One-click Configuration File Modification">
  ` curl -fsSL
  https://raw.githubusercontent.com/QuantumNous/new-api-docs/refs/heads/main/helper/factory-cli-setup.sh
  | bash `
</Callout>

![macos-img-03.webp](/assets/factory_droid_cli/macos_configure.png)

#### 3. Start Using Droid CLI

Now you can start using Droid CLI!

<Callout type="info" title="Launch Droid CLI">
  Launch Droid CLI directly: ` # Launch Droid CLI droid ` Use in a specific project: ` #
  Navigate to your project directory cd /path/to/your/project ` ` # Launch Droid CLI droid ` Press Enter to launch Droid CLI
</Callout>

> Droid CLI requires users to log in to an official account (free) before use.
