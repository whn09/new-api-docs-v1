## AIモデル設定方法

### Windows向け図解ガイド

#### 1. ターミナルを開く

![windows_open_terminal](/assets/factory_droid_cli/windows_open_terminal.png)

#### 2. Factory Droid CLIをインストール

公式ワンクリックインストールコマンド：

<Callout type="info" title="ワンクリックインストールコマンド">
  ` irm https://app.factory.ai/cli/windows | iex `
</Callout>

![windows-install](/assets/factory_droid_cli/windows_install_droid.webp)

#### 3. 設定ファイルを変更

Droid CLIがサードパーティAPIを使用するには、設定ファイルを変更する必要があります。

![windows-configure.webp](/assets/factory_droid_cli/factory_cli_setup_windows.png)

<Callout type="info" title="環境変数を変更">
  ` iex (irm
  'https://raw.githubusercontent.com/QuantumNous/new-api-docs/refs/heads/main/helper/factory-cli-setup.ps1')
  `
</Callout>

#### 4. Droid CLIの使用を開始

これでDroid CLIの使用を開始できます！

<Callout type="info" title="Droid CLIを起動">
  Droid CLIを直接起動: ` # 启动 Droid CLI droid ` 特定のプロジェクトで使用: ` #
  プロジェクトディレクトリに移動 cd C:\path\to\your\project ` ` # 启动 Droid CLI droid ` Enter
  でDroid CLIを起動
</Callout>

Droid CLIを使用するには、公式アカウント（無料）にログインする必要があります。

#### 5. Windowsのよくある問題解決

<Callout type="info" title="インストール時のプロンプト" collapsible>
  「permission denied」エラー。これは通常、権限の問題です。以下の解決策を試してください：
</Callout>

    - `PowerShell`を管理者として実行する

    - または`npm`をユーザーディレクトリを使用するように設定する：`npm config set prefix %APPDATA%\npm`

<Callout type="info" title="PowerShell実行ポリシーエラー" collapsible>
  実行ポリシーの制限に遭遇した場合は、以下を実行してください： ` Set-ExecutionPolicy -ExecutionPolicy
  RemoteSigned -Scope CurrentUser `
</Callout>

### macOS/Linux向け図解ガイド

#### 1. Droid CLIをインストール

<Callout type="info" title="Droid CLIをインストール">
  ターミナルを開き、以下のコマンドを実行してください： ` curl -fsSL https://app.factory.ai/cli | sh `
</Callout>

![macos-open-terminal](/assets/factory_droid_cli/macos_install_droid.webp)

![macos-img-02.webp](/assets/factory_droid_cli/macos_install_droid_2.webp)

インストールプロンプトに従って環境変数を変更してください（インストールプロンプトのコードを直接コピーしてください）：

Linuxでは状況に応じて `~/.bashrc` または `~/.zshrc` を選択してください

<Callout type="info" title="Droid CLI 環境変数（例としてのみ）">
  ` echo 'export PATH=/Users/ここを変更/.local/bin:$PATH' >> ~/.zshrc ` ` source
  ~/.zshrc `
</Callout>

#### 2. 設定ファイルを変更

Droid CLIがサードパーティAPIを使用するには、設定ファイルを変更する必要があります。

<Callout type="info" title="ワンクリックで設定ファイルを変更">
  ` curl -fsSL
  https://raw.githubusercontent.com/QuantumNous/new-api-docs/refs/heads/main/helper/factory-cli-setup.sh
  | bash `
</Callout>

![macos-img-03.webp](/assets/factory_droid_cli/macos_configure.png)

#### 3. Droid CLIの使用を開始

これでDroid CLIの使用を開始できます！

<Callout type="info" title="Droid CLIを起動">
  Droid CLIを直接起動: ` # 启动 Droid CLI droid ` 特定のプロジェクトで使用: ` #
  プロジェクトディレクトリに移動 cd /path/to/your/project ` ` # 启动 Droid CLI droid ` Enter
  でDroid CLIを起動
</Callout>

> Droid CLIを使用するには、公式アカウント（無料）にログインする必要があります。
